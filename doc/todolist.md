# CoursePilot 实施 Todo List

## 1. 使用说明

Todo 按可运行的纵向增量排序。`P0` 为 MVP 必须，`P1` 为首版质量闭环，`P2` 为增强项。每项只有在验收证据满足时才能勾选；需求编号来自 `system-requirements.md`。

## 2. 里程碑

| 里程碑 | 可演示结果 | 完成条件 |
|---|---|---|
| M0 工程骨架 | 应用可启动、配置可校验 | T001～T005 |
| M1 资料可检索 | 当前课程 Markdown/纯文本可上传并按章节检索 | T101～T108 |
| M2 Agent 最小闭环 | 四类任务可路由并输出结构化结果 | T201～T211 |
| M3 业务闭环 | 多题目各自的共享答案、评审、修改可持久化和版本化 | T301～T308 |
| M4 可交付 | UI、trace、错误处理和测试达到验收标准 | T401～T411 |

- [x] **T106 [P0] 实现全文预算策略**；关联 FR-005、NFR-003。验收：小资料全文返回，超限时按章节关键词排序。
- [x] **T107 [P0] 实现 `search_current_course`**；关联 FR-010、FR-012、NFR-001。验收：所有调用强制包含 active_course_id，调用参数不能覆盖它。

## 6. M3：多题目与每题答案版本闭环

- [x] **T301 [P0] 实现 TeamService/Repository**；关联 FR-007、FR-032。验收：可初始化一个小组并维护成员，第二个小组创建请求被拒绝。
- [x] **T302 [P0] 实现多题目 AssignmentService/Repository**；关联 FR-008、FR-032。验收：可创建、列出、更新和切换题目，首题自动激活，题目 ID 唯一。
- [x] **T303 [P0] 实现按题隔离的 AnswerService/Repository**；关联 FR-009、FR-032、FR-033。验收：每题共享答案版本独立、单调递增且只有一个当前版本，并记录操作成员。
- [x] **T304 [P0] 实现按题隔离的 ReviewService/Repository**；关联 FR-032。验收：评审 YAML 关联到具体题目的明确答案版本，可无损恢复为 Pydantic 模型。
- [x] **T305 [P0] 实现 RevisionService 事务**；关联 FR-009、FR-033、FR-034。验收：revision 与新的共享答案版本同时成功；任一步失败全部回滚。
- [x] **T306 [P0] 实现 ContextService**；关联 FR-030、FR-034。验收：小组、当前题目、课程、该题当前答案、最近评审和版本在切换后保持一致。

- [x] **T308 [P1] 增加共享答案版本对比和未解决问题检查**；关联 FR-029、FR-033。验收：修改结果展示操作成员和变更摘要，严重问题逐项标记已解决/未解决。

## 9. 发布门禁

首版发布前必须满足：

- 所有 P0 Todo 完成。
- `system-requirements.md` 第 7 章 7 项验收标准均有证据。
- 当前课程过滤测试通过率 100%，历史检索均有合法原因记录。
- 仓库和日志扫描未发现 API Key。
- 已知缺陷有明确影响、规避方案和处理优先级。
