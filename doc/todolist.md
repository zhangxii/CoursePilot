# CoursePilot 实施 Todo List

## 1. 使用说明

Todo 按可运行的纵向增量排序。`P0` 为 MVP 必须，`P1` 为首版质量闭环，`P2` 为增强项。每项只有在验收证据满足时才能勾选；需求编号来自 `system-requirements.md`。

## 2. 里程碑

| 里程碑 | 可演示结果 | 完成条件 |
|---|---|---|
| M0 工程骨架 | 应用可启动、配置可校验 | T001～T005 |
| M1 资料可检索 | 当前课程 PDF/PPTX 可导入并带页码检索 | T101～T108 |
| M2 Agent 最小闭环 | 四类任务可路由并输出结构化结果 | T201～T211 |
| M3 业务闭环 | 唯一小组大作业的共享答案、评审、修改可持久化和版本化 | T301～T308 |
| M4 可交付 | UI、trace、错误处理和测试达到验收标准 | T401～T411 |

## 6. M3：单一小组大作业与版本闭环

- [ ] **T301 [P0] 实现 TeamService/Repository**；关联 FR-007、FR-032。验收：可初始化一个小组并维护成员，第二个小组创建请求被拒绝。
- [ ] **T302 [P0] 实现 Singleton AssignmentService/Repository**；关联 FR-008、FR-032。验收：大作业可初始化、读取和更新题目/评分标准，第二份作业创建请求被拒绝且课程切换不创建新作业。
- [ ] **T303 [P0] 实现 AnswerService/Repository**；关联 FR-009、FR-032、FR-033。验收：唯一大作业的共享答案版本单调递增且唯一，并记录操作成员。
- [ ] **T304 [P0] 实现 ReviewService/Repository**；关联 FR-032。验收：评审关联到唯一大作业的明确答案版本，JSON 可无损恢复为 Pydantic 模型。
- [ ] **T305 [P0] 实现 RevisionService 事务**；关联 FR-009、FR-033、FR-034。验收：revision 与新的共享答案版本同时成功；任一步失败全部回滚。
- [ ] **T306 [P0] 实现 ContextService**；关联 FR-030、FR-034。验收：小组、唯一大作业、课程、当前答案、最近评审和版本在连续操作后保持一致。

## 9. 发布门禁

首版发布前必须满足：

- 所有 P0 Todo 完成。
- `system-requirements.md` 第 7 章 7 项验收标准均有证据。
- 当前课程过滤测试通过率 100%，历史检索均有合法原因记录。
- 仓库和日志扫描未发现 API Key。
- 已知缺陷有明确影响、规避方案和处理优先级。
