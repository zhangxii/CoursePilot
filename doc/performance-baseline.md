# CoursePilot 性能基线

## 1. 当前实现基线（M2–M4）

基线环境：Windows 11、Python 3.12、本地资料检索和离线模型替身。网络模型延迟不计入，目的是
固定编排调用上界；真实演示运行应通过应用日志补充端到端耗时。

| 请求 | 专业 Agent 调用数 | 默认检索范围 | 超预算章节上限 | 历史检索 |
|---|---:|---|---:|---|
| 总结 | 1 | current | 5 | 否 |
| 完成作业 | 1 | current | 5 | 否 |
| 评审 | 1 | current | 5 | 否 |
| 修改（已有评审） | 1 | current | 5 | 否 |
| 修改（无评审） | 2（review → revision） | current | 每次 5 | 否 |

2026-07-17 在本机运行 `python scripts/benchmark.py` 的离线编排实测：

| 请求 | 专业 Agent 调用数 | 检索结果数（替身） | 总耗时 |
|---|---:|---:|---:|
| 总结 | 1 | 0 | 0.629 ms |
| 完成作业 | 1 | 0 | 0.168 ms |
| 评审 | 1 | 0 | 0.080 ms |
| 修改（无评审） | 2 | 0 | 0.120 ms |

本地完整测试基线：44 个以上测试约 10 秒内完成（具体机器会波动）。普通请求没有无条件历史
检索；历史检索仅在四个 `ArchiveSearchReason` 之一被明确记录时允许。

## 2. 本轮扩展的调用上界目标（M5–M8）

生成和优化新增独立自动审查，因此不能继续使用旧表中的单 Agent 调用上界。实现后应重新运行基准并将实测结果补入本节。

| 请求 | 目标专业 Agent 调用上界 | 正式版本写入 | 说明 |
|---|---:|---:|---|
| 生成初版候选稿 | 4 | 0 | assignment → review → 可选 correction → review；修正最多一次 |
| 有明确方向的优化 | 4 | 0 | revision → review → 可选 correction → review；修正最多一次 |
| 无方向的问题分析 | 1 | 0 | 只分析并等待用户选择，不生成候选稿 |
| 正式评审 | 1 | 0 | 不修改答案 |
| 采用候选稿 | 0 | 1 | 纯本地事务，不调用模型 |
| 上传用户正式稿 | 0 | 1 | 纯本地解析与版本事务，不调用模型 |

性能验证还应记录：`.docx` 提取耗时、对话数量增加后的恢复耗时、候选稿差异计算耗时，以及生成/审查/修正各阶段的独立耗时。自动审查不可为了满足旧性能指标而跳过。
